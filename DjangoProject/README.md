# SmartPoultry

SmartPoultry — легковесная ERP-система для птицефабрики с фокусом на автоматизацию рутины, ранние инциденты и понятные операционные метрики.

## 1. Быстрый старт

```bash
cd /Users/dmitrykezarev/PycharmProjects/DjangoProject
python -m venv .venv
./.venv/bin/pip install -r requirements.txt
./.venv/bin/python manage.py migrate
./.venv/bin/python manage.py runserver
```

Открыть: [http://127.0.0.1:8000](http://127.0.0.1:8000)

## Прод-деплой на общий VDS

Инструкция для запуска этого Django-проекта как второго приложения на том же сервере:

- [deploy/DEPLOY_ON_SHARED_VDS.md](deploy/DEPLOY_ON_SHARED_VDS.md)


## 2. Роли и доступы

- **Owner**: полный доступ, управление менеджерами.
- **Manager**: единая роль с гибкой матрицей прав (`view/add/change/delete`) по сущностям и операциям.

## 3. Модульная структура

### 3.1 Дашборд

Содержит:
- KPI-карточки (корпуса, стада, поголовье, инциденты, прибыль, продажи, дебиторка, яйценоскость),
- графики (факт/эталон, прибыль, остаток корма в днях),
- последние инциденты и остатки.

Как считается:
- эффективность = `(яйца / живое поголовье) * 100`;
- живое поголовье = начальное поголовье − накопленный падеж.

### 3.2 Яйценоскость

Содержит:
- раздел “Яйценоскость по корпусам и стадам”,
- факт vs эталон по выбранной дате,
- настройка эталона.

Упрощение эталона:
- эталон задается как **яиц на 100 голов/день**;
- ввод по **этапам (диапазон недель)**, а не вручную по каждой неделе.

### 3.3 Склад

Содержит:
- остатки корма,
- ручной приход,
- корректировку остатков,
- журнал движений,
- Excel-экспорт,
- “Приход по заявке”.

Автоматизация:
- автосписание по формуле `живое поголовье × норма г/гол/день / 1000`;
- автоматическая заявка при низком прогнозе остатка;
- порог остатка настраивается в “Настройки автосписания”.

### 3.4 Заявки на закупку

Содержит:
- список/фильтры/экспорт PDF/печать,
- подтверждение или отклонение,
- ручное создание заявки,
- приемка поставки по утвержденной заявке с авто-закрытием.

### 3.5 Продажи

Содержит:
- список продаж и блок незавершенных продаж,
- смена статуса прямо в списке,
- PDF-экспорт.

Статусы:
- `Новая`,
- `В работе`,
- `Частично оплачена`,
- `Просрочка`,
- `Завершена`,
- `Отменена`.

### 3.6 Клиенты

Содержит:
- список клиентов,
- создание,
- редактирование.

### 3.7 Инциденты + CAPA

Содержит:
- список инцидентов с фильтрами,
- смену статуса,
- CAPA-поля (причина, действие, ответственный, срок),
- PDF-выгрузки (реестр и карточка инцидента),
- справочник причин инцидентов.

### 3.8 KPI руководителя

Содержит свод по корпусам за 14 дней:
- средняя эффективность,
- средний бой,
- корм на голову,
- выручка по яйцу,
- расход на корм,
- маржа,
- количество дней с фактическими данными.

Формула маржи:
- `маржа = выручка по яйцу − расход на корм`.

### 3.9 Администрирование

Содержит:
- менеджеры (создание, редактирование, права),
- корпуса (привязка ответственного менеджера, активность),
- настройки автоматизации (единая точка порогов).

## 4. Автоматические инциденты и сценарии

### 4.1 Инциденты по дневной записи (post_save)

1. **Падение яйценоскости** (`egg_drop`)  
   Триггер: факт ниже эталона на заданный порог.

2. **Повышенный бой** (`breakage`)  
   Триггер: бой выше заданного порога.

3. **Производственная аномалия** (`production_anomaly`)  
   Триггер: падение сбора против скользящей средней или всплеск падежа.

4. **Низкий остаток корма** (`feed_stock`)  
   Триггер: прогноз дней до нуля ниже порога.

5. **Плановая замена стада** (`flock_rotation`)  
   Триггер: прогноз маржинальности через N дней уходит ниже порога.

### 4.2 Ежедневная фоновая проверка дебиторки

Команда:
```bash
./.venv/bin/python manage.py check_overdue_sales
```

Что делает:
- находит продажи со статусом `Частично оплачена`, старше лимита дней,
- переводит в `Просрочка`,
- генерирует PDF-уведомление о доплате в `generated_reports/`,
- создает инцидент `Просроченная дебиторка`.

## 5. Настройки автоматизации (без кода)

Экран: `Администрирование -> Настройки автоматизации`.

Поля:
- порог падения яйценоскости,
- порог боя,
- порог аномального дневного падения,
- множитель нормы падежа,
- порог падежа от живого поголовья,
- горизонт прогноза ротации,
- минимум истории для прогноза,
- порог маржинальности для сигнала ротации,
- срок просрочки дебиторки.

Все значения применяются сразу к новым проверкам.
